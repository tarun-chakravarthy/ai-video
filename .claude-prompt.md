# AI Video Dashboard - Claude Code Prompt

You are assisting with development of an AI Video Dashboard built with:
- **Next.js 16.2.9** with React 19
- **TypeScript** (strict mode)
- **Tailwind CSS 4** with shadcn/ui
- **Video editing features** (timeline, clips, preview, AI suggestions)

## Your Role
Generate clean, functional, production-ready code that:
1. Passes `pnpm build` without errors
2. Has zero TypeScript errors
3. Follows React best practices
4. No cascading renders or hydration mismatches
5. Uses proper TypeScript typing (no `any`)

## Key Constraints

### ✅ DO
- Use TypeScript interfaces for all component props and refs
- Export types that other components need
- Write effects that only have side effects (listeners, subscriptions)
- Use `useMemo` for derived state (don't use setState in effects)
- Use `useCallback` for event handlers passed as props
- Clean up all listeners/subscriptions in effect cleanup functions
- Initialize state with safe defaults, sync post-hydration
- Cast event targets properly: `(e.target as HTMLInputElement)`
- Test that code compiles before finalizing

### ❌ DON'T
- Use `any` types - always define proper interfaces
- Call `setState` directly in effect bodies
- Add state to dependency arrays that cause cascading renders
- Use `typeof window === 'undefined'` in state initializers
- Create functions/objects inside render bodies
- Forget to return cleanup functions from effects
- Skip TypeScript type definitions
- Mix controlled and uncontrolled components

## Example Good Pattern
```typescript
// video-preview.tsx
interface VideoPreviewHandle {
  play: () => Promise<void>;
  pause: () => void;
  seek: (time: number) => void;
}

interface VideoPreviewProps {
  videoUrl?: string | null;
  onTimeUpdate?: (time: number, duration: number) => void;
}

export const VideoPreview = forwardRef<VideoPreviewHandle, VideoPreviewProps>(
  ({ videoUrl, onTimeUpdate }, ref) => {
    const videoRef = useRef<HTMLVideoElement>(null);
    const [isPlaying, setIsPlaying] = useState(false);

    useImperativeHandle(ref, () => ({
      play: async () => { await videoRef.current?.play(); },
      pause: () => { videoRef.current?.pause(); },
      seek: (time) => { if (videoRef.current) videoRef.current.currentTime = time; }
    }));

    useEffect(() => {
      const video = videoRef.current;
      if (!video) return;

      const handleTimeUpdate = () => {
        onTimeUpdate?.(video.currentTime, video.duration);
      };

      video.addEventListener('timeupdate', handleTimeUpdate);
      return () => video.removeEventListener('timeupdate', handleTimeUpdate);
    }, [onTimeUpdate]);

    return <video ref={videoRef} src={videoUrl} />;
  }
);
```

## Common Tasks

### Adding a new component
1. Create file with `PascalCase.tsx` name
2. Define props interface
3. Export component with proper typing
4. Use in parent with TypeScript support

### Fixing TypeScript errors
- Always cast event targets: `(e.target as HTMLInputElement)`
- Define interfaces for complex types
- Use `React.RefObject<T>` for refs
- Export shared types so other components can import

### Handling state updates
- Initialize with safe defaults
- Derive values with `useMemo` instead of state + effects
- Effects should only handle side effects
- Keep effect dependencies minimal
