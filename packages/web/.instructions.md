# Claude Code Execution Guidelines for AI Video Dashboard

## Project Context
- **Framework**: Next.js 16.2.9 with React 19
- **Styling**: Tailwind CSS 4 + shadcn/ui components
- **Type Safety**: Full TypeScript with strict mode
- **Video Editing**: Custom video player, timeline, clip editing, AI suggestions

## Code Quality Standards

### TypeScript Requirements
- ✅ **Strict typing** - No `any` types. Always define interfaces/types
- ✅ **Export interfaces** - Export types used across components
- ✅ **Proper generics** - Use `React.ReactNode`, `React.RefObject<T>`, etc.
- ✅ **Event typing** - Use `React.ChangeEvent<HTMLInputElement>`, `KeyboardEvent`, etc.

### React Best Practices
- ✅ **No cascading setState** - Never call setState directly in effect bodies
- ✅ **Optimize effects** - Minimize dependencies, use `useMemo` for derived state
- ✅ **Use callbacks** - Wrap complex handlers in `useCallback` to prevent re-renders
- ✅ **Avoid hydration mismatches** - No `typeof window === 'undefined'` in state initializers
- ✅ **No memory leaks** - Always clean up listeners, intervals, and subscriptions
- ✅ **Default values first** - Initialize state with safe defaults, sync after hydration

### Component Structure
```typescript
// ✅ Good pattern
interface ComponentProps {
  // Props types
}

export function MyComponent({ prop }: ComponentProps) {
  // State
  const [state, setState] = useState(false);
  
  // Derived values (no effects needed)
  const derivedValue = useMemo(() => {
    return compute(state);
  }, [state]);
  
  // Effects (listeners, subscriptions)
  useEffect(() => {
    const handler = () => {
      // Handle external event
    };
    window.addEventListener('event', handler);
    return () => window.removeEventListener('event', handler);
  }, []);
  
  return <div>...</div>;
}
```

### File Naming
- Components: `PascalCase.tsx` (e.g., `VideoPlayer.tsx`)
- Utilities: `camelCase.ts` (e.g., `videoUtils.ts`)
- Folders: lowercase with hyphens (e.g., `components/ui/`)

## Common Patterns

### Managing Refs
```typescript
import { useRef, useImperativeHandle } from 'react';

export interface VideoPreviewHandle {
  play: () => Promise<void>;
  pause: () => void;
}

export const VideoPreview = forwardRef<VideoPreviewHandle, Props>(
  (props, ref) => {
    const videoRef = useRef<HTMLVideoElement>(null);
    
    useImperativeHandle(ref, () => ({
      play: async () => { /* ... */ },
      pause: () => { /* ... */ }
    }));
    
    return <video ref={videoRef} />;
  }
);
```

### Handling Events
```typescript
// ✅ Good: Properly typed
const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
  const value = parseFloat(e.target.value);
};

// ❌ Bad: Using 'any' or casting
const handleChange = (e: any) => { };
```

### Avoiding Cascading Renders
```typescript
// ✅ Good: Separate independent effects
useEffect(() => {
  // Effect 1: Sidebar init (only runs once)
  setIsSidebarOpen(window.innerWidth >= 1024);
}, []);

useEffect(() => {
  // Effect 2: Resize listener (independent)
  const handleResize = () => { /* ... */ };
  window.addEventListener('resize', handleResize);
  return () => window.removeEventListener('resize', handleResize);
}, []);

// ❌ Bad: Cascading state changes
const [isHydrated, setIsHydrated] = useState(false);

useEffect(() => {
  setIsSidebarOpen(true);
  setIsHydrated(true); // This triggers other effects!
}, []);

useEffect(() => {
  if (!isHydrated) return; // Runs when isHydrated changes
  // This creates cascading renders
}, [isHydrated]);
```

## When Making Changes

1. **Read the current code** first to understand patterns
2. **Follow existing conventions** - Match the codebase style
3. **Test locally** - Run `pnpm dev` and verify in browser
4. **Build before committing** - Ensure `pnpm build` passes
5. **Commit with clear message** - Describe what changed
6. **Push to GitHub** - Keep remote synchronized

## Dashboard Library Integration (If Needed)

When adding dashboard libraries:
- Use shadcn/ui components first (already installed)
- Install with `npx shadcn-ui@latest add [component]`

## Error Handling
- Always wrap async operations in try/catch
- Log errors to console for development
- Show user-friendly error messages
- Never silently fail

## Performance Guidelines
- Use `useMemo` for expensive computations
- Use `useCallback` for event handlers passed as props
- Avoid creating functions/objects in render
- Minimize effect dependencies
- Clean up listeners on unmount
